# 阿米巴单位时间核算表 — 使用手册

## 1. 应用简介

### 什么是阿米巴经营？

阿米巴经营（Amoeba Management）是日本京瓷公司创始人稻盛和夫创立的一套管理会计方法。它将企业划分为若干个称为"阿米巴"的小型独立核算单位，每个阿米巴像一个微型公司一样独立核算收入、费用和附加值，从而实现全员参与经营、实时掌握经营状况。

**单位时间附加值**是阿米巴经营的核心指标，它衡量的是"每小时劳动创造了多少附加价值"，直观反映每个阿米巴的经营效率。

### 核心功能

本应用是阿米巴单位时间核算表的数字化工具，提供以下功能：

- **阿米巴管理**：创建和管理多个阿米巴组织（生产型、营销型、研发型、管理型）
- **数据录入**：录入收入明细、费用明细、劳动时间，自动计算 9 大核算指标
- **核算报表**：按周期查看详细核算报表，支持上期对比分析
- **趋势分析**：通过折线图对比多个阿米巴的关键指标变化趋势
- **Excel 导出**：一键导出包含 4 个工作表的专业格式 Excel 报表
- **类别自定义**：每个阿米巴可自定义收入和费用类别

### 技术信息

本应用基于 Tauri 2.x 框架构建，使用 Vue 3 前端界面和 Rust 后端服务，数据存储在本地 SQLite 数据库中。支持 Windows、macOS 和 Linux 平台。

## 2. 安装与启动

### 支持平台

| 平台 | 最低版本 |
|------|----------|
| Windows | Windows 10 及以上 |
| macOS | macOS 11 及以上 |
| Linux | 主流发行版（Ubuntu 20.04+、Debian 11+ 等） |

### 安装步骤

1. 从发布页面下载对应平台的安装包：
   - Windows：`.msi` 或 `.exe` 安装程序
   - macOS：`.dmg` 磁盘映像
   - Linux：`.deb` 或 `.AppImage` 安装包
2. 运行安装程序，按照提示完成安装。
3. 安装完成后，从开始菜单（Windows）、应用程序文件夹（macOS）或应用菜单（Linux）启动应用。

### 数据存储位置

应用的所有数据存储在操作系统的用户数据目录中，文件名为 `amoeba.db`（SQLite 数据库）。

| 平台 | 数据路径 |
|------|----------|
| Windows | `C:\Users\<用户名>\AppData\Roaming\cn.shinehe.accounting\amoeba.db` |
| macOS | `~/Library/Application Support/cn.shinehe.accounting/amoeba.db` |
| Linux | `~/.local/share/cn.shinehe.accounting/amoeba.db` |

> **提示**：如需备份数据，只需复制 `amoeba.db` 文件即可。如需迁移数据，将该文件复制到新设备的对应目录即可。

## 4. 功能详解

### 4.1 首页仪表盘

首页仪表盘是应用启动后显示的默认页面，提供所有阿米巴的快速概览。

**顶部统计卡片**：
- **阿米巴总数**：系统中已创建的全部阿米巴数量
- **本月总销售额**：所有阿米巴最新核算记录的总销售额之和
- **平均单位时间附加值**：所有阿米巴单位时间附加值的平均值（排除无数据的阿米巴）
- **平均附加值率**：所有阿米巴附加值率的平均值（排除无数据的阿米巴）

**各阿米巴最新核算结果表格**：
列出每个阿米巴的最新核算数据，包括阿米巴名称、负责人、核算期间、总销售额、总费用、附加价值、单位时间附加值和附加值率。负值以红色显示，方便快速识别经营异常。

> 仪表盘数据为只读概览，点击侧边栏进入具体功能页面可进行详细操作。

### 4.2 阿米巴管理

在此页面管理所有阿米巴组织单位。

**阿米巴列表**：表格显示所有已创建的阿米巴，包含名称、类型、负责人、状态（启用/停用）、创建时间。

#### 创建阿米巴

1. 点击**新增阿米巴**按钮
2. 在弹出的对话框中填写以下信息：
   - **名称**（必填）：阿米巴单位的名称，如"制造一科"、"销售部"
   - **类型**（必选）：从下拉框选择一种类型
   - **负责人**（必填）：该阿米巴的负责人姓名
3. 点击**确定**完成创建

#### 四种阿米巴类型

| 类型 | 说明 | 典型应用 |
|------|------|----------|
| 生产型 | 负责产品生产制造的单位 | 制造车间、组装线、加工班组 |
| 营销型 | 负责销售和市场推广的单位 | 销售团队、市场部、门店 |
| 研发型 | 负责技术研发和产品开发的单位 | 研发小组、设计团队 |
| 管理型 | 负责行政管理和支持服务的单位 | 人事、财务、IT 支持 |

#### 编辑与删除

- **编辑**：点击表格中某行的**编辑**按钮，修改信息后点击确定。
- **删除**：点击**删除**按钮，确认后将该阿米巴及其关联的所有核算数据永久删除。此操作不可恢复，请谨慎操作。

### 4.3 数据录入

数据录入是应用的核心功能页面，用于录入和提交核算数据。

页面分为"新建"模式（路径 `/entry`）和"编辑"模式（路径 `/entry/:id`），编辑模式会自动加载已有数据进行修改。

#### 表单分为五个区域：

**① 基本信息**
- **阿米巴**：从下拉框选择要核算的阿米巴（编辑模式下不可更改）
- **核算周期**：选择"按月"、"按周"或"按日"
- **日期范围**：选择核算的起止日期，格式为 YYYY-MM-DD
- **备注**（可选）：填写说明文字

**② 收入明细**

每行收入明细包含三个字段：
- **收入类别**：从下拉框选择收入类别（如对外销售、内部交易、服务收入等）
- **金额**：输入收入金额（数字，支持 2 位小数）
- **说明**（可选）：对该项收入的文字说明

点击行尾的**删除**按钮可移除该行。点击**添加自定义收入**按钮可增加新的收入行。

底部显示**收入合计**金额。

点击标题栏右侧的"管理收入类别"按钮，可在弹窗中增删改收入类别。

**③ 费用明细**

每行费用明细包含三个字段：
- **费用类别**：从下拉框选择费用类别（如原材料费、电费、折旧费等）
- **金额**：输入费用金额（数字，支持 2 位小数）
- **说明**（可选）：对该项费用的文字说明

操作方式与收入明细相同。底部显示**费用合计**金额。

点击标题栏右侧的"管理费用类别"按钮，可在弹窗中增删改费用类别。

**④ 劳动时间**

四个输入字段，采用 2×2 网格布局：
- **正常工作时间**（小时）：正常工作时段的总工时，支持 1 位小数
- **加班时间**（小时）：加班时段的总工时，支持 1 位小数
- **公共时间分摊**（小时）：需要分摊的公共时间（如会议、培训等），支持 1 位小数
- **当期人数**：该阿米巴当期的人员数量（整数，最小为 1）

底部显示**总劳动时间**（= 正常时间 + 加班时间 + 公共时间分摊）。

**⑤ 核算结果预览**

根据输入的收入、费用和工时数据，实时计算并显示 9 大核算指标：

| 指标 | 说明 |
|------|------|
| 总销售额 | 所有收入金额之和 |
| 总费用 | 所有费用金额之和 |
| 附加价值 | 总销售额 − 总费用 |
| 总劳动时间 | 正常时间 + 加班时间 + 公共时间 |
| **单位时间附加值** | 附加价值 ÷ 总劳动时间（**核心 KPI**） |
| 人均销售额 | 总销售额 ÷ 人数 |
| 人均附加值 | 附加价值 ÷ 人数 |
| 附加值率 | 附加价值 ÷ 总销售额 × 100% |
| 费用率 | 总费用 ÷ 总销售额 × 100% |

其中"单位时间附加值"以黄色背景高亮显示，是该页面的核心关注指标。负值以红色显示。

#### 底部操作

- **取消**：放弃当前编辑，返回首页
- **重置**：清空所有已填写的内容，恢复到初始状态
- **保存**：验证所有输入并提交保存。保存成功后自动返回首页。

### 4.4 类别设置

此页面用于为每个阿米巴独立配置收入和费用类别。

#### 操作步骤

1. 从顶部下拉框选择一个阿米巴
2. **收入类别**区域：在可编辑表格中增删改收入类别。每行包含名称、说明、排序（数字越小越靠前）。
3. **费用类别**区域：同上，管理费用类别。
4. 修改后**保存设置**按钮变为可用，点击保存。

#### 恢复默认

点击**恢复默认**按钮，系统会清除自定义类别并恢复系统预设的默认类别：
- 4 个默认收入类别：对外销售、内部交易、服务收入、其他收入
- 10 个默认费用类别：原材料费、外部加工费、电费、折旧费、运输费、维修费、办公费、通讯费、差旅费、其他费用

> **注意**：切换阿米巴时如有未保存的修改，系统会提示确认。恢复默认操作不可撤销。

### 4.5 核算报表

查看指定阿米巴的详细核算报表，支持上期对比分析。

**筛选条件**：
- **阿米巴**：选择要查看的阿米巴（可清除选择查看所有阿米巴）
- **周期类型**：按月 / 按周 / 按日（选项卡切换）
- **日期范围**：指定要查看的时间区间
- **显示上期对比**：开关按钮，默认开启

**报表内容**：

报表以表格形式展示，按以下结构分组：

1. **销售收入**：按类别列出每项收入金额，含小计行（绿色背景）
2. **费用明细**：按类别列出每项费用金额，含小计行（绿色背景）
3. **附加价值**：附加价值金额及附加值率（百分比）
4. **劳动时间**：正常工时、加班工时、公共工时、总工时、当期人数
5. **核心指标**（黄色背景）：单位时间附加值、人均销售额、人均附加值、费用率

**上期对比功能**：

开启后，表格在"本期"列旁额外显示"上期"、"增减"和"增减率"列。正值以绿色显示，负值以红色显示（费用率的颜色逻辑相反，费用上升为红色）。

### 4.6 趋势分析

通过折线图直观展示关键指标随时间的变化趋势。

**筛选条件**：
- **阿米巴**：多选下拉框，可同时选择多个阿米巴进行对比
- **指标**：选择要分析的指标：
  - 单位时间附加值（核心效率指标）
  - 附加值率（盈利能力指标）
  - 费用率（成本控制指标）
- **周期**：按月 / 按周 / 按日

**图表交互**：
- 鼠标悬停数据点可查看具体数值
- 顶部图例展示各阿米巴对应的折线颜色，点击图例可切换显示/隐藏
- 折线平滑渲染，Y 轴对百分比指标自动添加 `%` 后缀

## 5. 核心概念与公式

### 9 大核算公式

本系统自动计算以下 9 个核心指标：

| # | 指标 | 公式 | 通俗解释 |
|---|------|------|----------|
| 1 | **总销售额** | 所有收入金额之和 | 该阿米巴在一定时期内创造的收入总额 |
| 2 | **总费用** | 所有费用金额之和 | 该阿米巴在一定时期内消耗的成本总额 |
| 3 | **附加价值** | 总销售额 − 总费用 | 阿米巴创造的净价值，即收入减去成本后的毛利 |
| 4 | **总劳动时间** | 正常工时 + 加班工时 + 公共工时 | 全体员工投入的总工作小时数 |
| 5 | **单位时间附加值** | 附加价值 ÷ 总劳动时间 | **核心 KPI**：每小时劳动创造了多少附加价值。数值越高，效率越高 |
| 6 | **人均销售额** | 总销售额 ÷ 人数 | 平均每人创造的销售额 |
| 7 | **人均附加值** | 附加价值 ÷ 人数 | 平均每人创造的附加价值 |
| 8 | **附加值率** | (附加价值 ÷ 总销售额) × 100% | 销售额中有多少比例转化为附加价值，反映盈利效率 |
| 9 | **费用率** | (总费用 ÷ 总销售额) × 100% | 销售额中有多少比例被费用消耗，反映成本控制水平 |

> 附加值率 + 费用率 = 100%，两者互为补充。

### 关键指标解读

**单位时间附加值**是最重要的经营指标。举例说明：

- 如果某阿米巴当月附加价值为 100,000 元，总劳动时间为 2,000 小时
- 则单位时间附加值为 50 元/小时
- 这意味着每小时劳动可以创造出 50 元的附加价值

管理者应关注单位时间附加值的变化趋势：
- **上升**：效率提高，经营改善
- **下降**：需要分析是收入减少、费用增加还是工时增多导致的

## 6. 常见问题

### Q：数据在哪里，如何备份？

数据存储在应用数据目录下的 `amoeba.db` 文件中。参见第 2 节的"数据存储位置"表格。备份时只需复制该文件，迁移到新设备时将其放入对应目录即可。

### Q：一个阿米巴可以有多少条核算记录？

无限制。同一阿米巴可创建多条核算记录，按日期范围区分。通常每个阿米巴每个核算周期（如每月）录入一条记录。

### Q：为什么核算结果预览中的数字和保存后看到的不一致？

前端预览和后端计算使用完全相同的公式。如出现差异，可能是输入数据在传输过程中有变化。如果确认存在不一致，请检查是否为版本更新导致。

### Q：如何编辑已保存的核算记录？

在核算报表页面找到要编辑的记录，通过 URL 路径 `/entry/:id` 进入编辑模式。也可以直接在数据录入页面，通过浏览器地址栏输入对应的记录 ID 访问。

### Q：删除阿米巴后数据能恢复吗？

删除阿米巴会级联删除其所有关联的核算记录、费用明细、工时数据和类别设置。此操作不可撤销，建议在删除前先导出 Excel 备份数据。

### Q：导出的 Excel 文件乱码怎么办？

Excel 文件使用 UTF-8 编码和标准 xlsx 格式，在 Microsoft Excel 2016+、WPS Office、LibreOffice 中均可正常打开。如遇乱码，请检查 Excel 版本是否过旧。

### Q：支持多用户同时使用吗？

当前版本为单用户本地应用，不支持多用户网络共享。如需多人协作，建议将数据文件放置在共享目录中，但请注意同一时间只能有一人编辑，否则可能出现数据冲突。

### Q："公共时间分摊"是什么？

公共时间是指不能直接归属于某个阿米巴的正常生产活动的间接时间，例如全员会议、公司培训、团建活动等。如果一个阿米巴的员工参与了这些活动，应将相应的时间分摊到该阿米巴的核算中。如果不需要，可以填 0。

### Q："上期"指的是什么？

在核算报表中，"上期"指的是同一阿米巴在相同周期类型下的前一条记录。例如，如果本期是 2024 年 3 月，上期就是 2024 年 2 月。
