# 阿米巴经营 — 单位时间核算表工具 设计文档

> **版本**：v2.0（Tauri 重构版）
> **日期**：2026-05-03
> **应用类型**：桌面应用（单用户，Tauri 架构）
> **核算粒度**：按月 / 按周 / 按日

---

## 1. 项目概述

### 1.1 背景

阿米巴经营是稻盛和夫创立的企业经营管理模式，其核心工具**单位时间核算表**用于衡量每个阿米巴（独立核算小组织）在单位时间内创造的附加价值。本工具旨在将该核心工具数字化，实现数据录入、自动计算、表格展示与 Excel 导出，帮助个人或小团队高效进行阿米巴经营核算。

### 1.2 核心公式

```
单位时间附加值 = （销售额 - 费用）÷ 总劳动时间
```

其中：
- **销售额** = 对外销售收入 + 内部交易收入
- **费用** = 原材料费 + 外部加工费 + 电费 + 折旧费 + 杂费 + 其他
- **总劳动时间** = 正常工作时间 + 加班时间 + 部门公共时间分摊

### 1.3 功能范围

| 功能模块 | 说明 |
|----------|------|
| **数据录入** | 录入阿米巴基础信息、销售额、费用明细、劳动时间 |
| **自动计算** | 根据录入数据自动计算附加值、单位时间附加值等指标 |
| **表格输出** | 以标准核算表格式展示计算结果，支持月/周/日三种维度 |
| **Excel 导出** | 将核算表导出为标准 Excel 文件，保留格式与公式 |

### 4.2 自动计算模块

#### 4.2.1 计算公式定义

```
┌──────────────────────────────────────────────────────────┐
│                    计算公式体系                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① 总销售额 = 对外销售额 + 内部交易额                      │
│                                                          │
│  ② 总费用 = Σ 各项费用明细金额                             │
│         = 原材料费 + 外部加工费 + 电费 + 折旧费            │
│           + 运输费 + 维修费 + 办公费 + 通讯费              │
│           + 差旅费 + 其他费用                              │
│                                                          │
│  ③ 附加价值 = 总销售额 - 总费用                            │
│                                                          │
│  ④ 总劳动时间 = 正常工作时间 + 加班时间 + 公共时间分摊      │
│                                                          │
│  ⑤ 单位时间附加值 = 附加价值 ÷ 总劳动时间                   │
│                                                          │
│  ⑥ 人均销售额 = 总销售额 ÷ 当期人数                        │
│                                                          │
│  ⑦ 人均附加值 = 附加价值 ÷ 当期人数                        │
│                                                          │
│  ⑧ 附加值率 = 附加价值 ÷ 总销售额 × 100%                   │
│                                                          │
│  ⑨ 费用率 = 总费用 ÷ 总销售额 × 100%                      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

#### 4.2.2 多维度核算逻辑

```
                    ┌─────────────┐
                    │  原始录入数据  │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  按日核算  │ │  按周核算  │ │  按月核算  │
        │  DAY      │ │  WEEK    │ │  MONTH   │
        └────┬─────┘ └────┬─────┘ └────┬─────┘
             │            │            │
             ▼            ▼            ▼
        日核算表       周核算表       月核算表
        (每日一条)     (每周汇总)     (每月汇总)
             │            │            │
             └────────────┼────────────┘
                          ▼
                   ┌─────────────┐
                   │  期间汇总    │
                   │  (周汇总日   │
                   │   月汇总周)  │
                   └─────────────┘
```

**汇总规则**：

| 维度 | 汇总方式 | 说明 |
|------|----------|------|
| **按日** | 直接取当日录入数据 | 每天一条核算记录 |
| **按周** | SUM(周一~周日) | 自动汇总一周内的日数据 |
| **按月** | SUM(1日~月末) | 自动汇总一月内的所有数据 |

> 注：周汇总基于自然周（周一至周日），月汇总基于自然月。

#### 4.2.3 计算流程

```
输入数据 → 校验 → 计算总销售额 → 计算总费用 → 计算附加价值
    → 计算总劳动时间 → 计算单位时间附加值 → 计算辅助指标
    → 存储结果 → 刷新展示
```

#### 4.2.4 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 总劳动时间为 0 | 提示"劳动时间不能为0"，阻止保存 |
| 附加价值为负数 | 正常计算，在表格中以红色标记 |
| 总销售额为 0 | 正常计算，附加值率和费用率显示为 "N/A" |
| 数据缺失（部分费用项未填） | 未填项默认为 0，在表格中灰色标记 |

## 6. 核心算法伪代码

### 6.1 核算计算引擎（Rust 后端）

```rust
/// 核心核算计算 — 在 Rust 后端执行，保证精度与性能
pub fn calculate_accounting(record: &AccountingRecord) -> AccountingResult {
    // 1. 计算总销售额
    let total_sales = record.external_sales + record.internal_sales;

    // 2. 计算总费用
    let total_expense: f64 = record
        .expense_details
        .iter()
        .map(|d| d.amount)
        .sum();

    // 3. 计算附加价值
    let added_value = total_sales - total_expense;

    // 4. 计算总劳动时间
    let total_hours = record.labor_time.normal_hours
        + record.labor_time.overtime_hours
        + record.labor_time.public_hours;

    // 5. 计算单位时间附加值（核心指标）
    let unit_value = if total_hours > 0.0 {
        added_value / total_hours
    } else {
        0.0
    };

    // 6. 计算辅助指标
    let headcount = record.labor_time.headcount.max(1) as f64;
    let sales_per_person = total_sales / headcount;
    let value_per_person = added_value / headcount;
    let value_rate = if total_sales > 0.0 {
        (added_value / total_sales) * 100.0
    } else {
        0.0
    };
    let expense_rate = if total_sales > 0.0 {
        (total_expense / total_sales) * 100.0
    } else {
        0.0
    };

    AccountingResult {
        total_sales,
        total_expense,
        added_value,
        total_hours,
        unit_value,
        sales_per_person,
        value_per_person,
        value_rate,
        expense_rate,
    }
}
```

### 6.2 Tauri IPC 命令定义（Rust）

```rust
/// Tauri Command：保存核算记录并自动计算
#[tauri::command]
async fn save_record(
    state: State<'_, AppState>,
    record: AccountingRecordInput,
) -> Result<AccountingResult, String> {
    // 1. 数据校验
    validate_record(&record).map_err(|e| e.to_string())?;

    // 2. 核算计算
    let result = calculate_accounting(&record);

    // 3. 持久化到 SQLite
    state
        .record_repo
        .insert(&record, &result)
        .await
        .map_err(|e| e.to_string())?;

}
```

### 6.3 前端实时预览计算（TypeScript）

```typescript
// composables/useAccounting.ts
// 前端轻量计算，用于录入时实时预览（不依赖后端）

export function useAccounting() {
  const calculate = (record: RecordInput): AccountingResult => {
    const totalSales = record.externalSales + record.internalSales
    const totalExpense = record.expenses.reduce((sum, e) => sum + e.amount, 0)
    const addedValue = totalSales - totalExpense
    const totalHours = record.labor.normalHours
      + record.labor.overtimeHours
      + record.labor.publicHours
    const unitValue = totalHours > 0 ? addedValue / totalHours : 0
    const headcount = Math.max(record.labor.headcount, 1)
    const valueRate = totalSales > 0 ? (addedValue / totalSales) * 100 : 0
    const expenseRate = totalSales > 0 ? (totalExpense / totalSales) * 100 : 0

    return {
      totalSales,
      totalExpense,
      addedValue,
      totalHours,
      unitValue,
      salesPerPerson: totalSales / headcount,
      valuePerPerson: addedValue / headcount,
      valueRate,
      expenseRate,
    }
  }

  return { calculate }
}
```

### 6.4 多维度汇总器（Rust）

```rust
/// 按指定维度汇总核算记录
pub fn aggregate_records(
    records: &[AccountingRecord],
    period_type: &str,
) -> Vec<AggregatedRecord> {
    if period_type == "day" {
        return records.iter().map(|r| {
            AggregatedRecord::from_record(r, &calculate_accounting(r))
        }).collect();
    }

    // 按周期分组（周 / 月）
    let grouped = group_by_period(records, period_type);

    grouped
        .into_iter()
        .map(|(period_key, group)| {
            let external_sales: f64 = group.iter().map(|r| r.external_sales).sum();
            let internal_sales: f64 = group.iter().map(|r| r.internal_sales).sum();
            let expenses = merge_expenses(&group);
            let labor = merge_labor_time(&group);

            let aggregated = AccountingRecord {
                external_sales,
                internal_sales,
                expense_details: expenses,
                labor_time: labor,
                period_type: period_type.to_string(),
                ..Default::default()
            };

            let result = calculate_accounting(&aggregated);
            AggregatedRecord::from_record(&aggregated, &result)
        })
        .collect()
}
```
